TEST SCENARIOS:

- Positive: Get all vehicle types
- Positive: Get vehicle models filtered by vehicle type
- Positive: Get locations filtered by type (branch)
- Positive: Get status codes for specific entity (order)
- Positive: Get states by country ID
- Positive: Get cities by state ID
- Negative: Get status codes without entity parameter (expect 400)
- Negative: Get states without countryId (expect 400)
- Negative: Get cities without stateId (expect 400)
- Negative: Access lookup without authentication (expect 401)
- Edge Case: Get vehicle models with non-existent vehicleTypeId (expect empty list)
- Edge Case: Get locations with invalid type filter (expect empty list)
- Edge Case: Get cities for state with no cities defined (expect empty list)

CHAINING:
- Extract vehicleTypeId from vehicle types response for vehicle models query
- Extract countryId from countries response for states query
- Extract stateId from states response for cities query
- Extract locationId from locations response for branch/hub specific operations
- Extract categoryId from product categories for inventory operations
- Extract roleId from user roles for user management operations
